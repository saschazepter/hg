Registration
============

All config options used within Mercurial must be registered.

* Core config options must be registered in mercurial/configitems.toml.

* Config options for core extensions must also be registered in
  configitems.toml, setting ``in_core_extension`` to the name of the extension.

* Config options for third-party extensions must be registered in code through
  the ``registrar`` API.

Registrar API
-------------

Here is an example of registering a config using the ``registrar`` API (also
used for commands and others)::

    configtable = {}
    configitem = registrar.configitem(configtable)

    configitem(b'blackbox', b'dirty', default=False)

If ``default`` is a callable, it is called to retrieve the default value when
accessing the config option. For example, ``default=list`` would make the config
option default to an empty list.

If ``default`` is ``mercurial.configitems.dynamicdefault``, then there is no
fixed default, and a default value is explicitly required when reading.
